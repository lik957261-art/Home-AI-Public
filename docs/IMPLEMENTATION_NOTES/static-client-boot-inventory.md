## Notes For Vite Migration

- Script order is part of the current classic shell behavior. If
  `public/index.html` script order changes, regenerate this inventory in the
  same change.
- `Produced globals` and `Consumed globals` are heuristic static-analysis
  fields used to plan ES-module extraction. They must be verified when moving
  each module.
- `Runtime Facade Candidate Uses` identifies symbols that should move behind
  an explicit client runtime facade before bundling the primary shell.
- `Import side effects` marks files that may need mount functions or
